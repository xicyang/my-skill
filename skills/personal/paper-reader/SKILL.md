# 论文阅读笔记生成器

用户提供arxiv论文链接后，通读论文全文并生成结构化的markdown阅读笔记。

## 触发条件

用户提供arxiv论文URL（如 `https://arxiv.org/abs/XXXX.XXXXX` 或 `https://arxiv.org/pdf/XXXX.XXXXX`），并要求生成阅读笔记、论文总结、文献笔记等。

## 语言要求

笔记内容使用中文撰写，论文标题、方法名、术语等保留英文原文。

## 笔记结构

生成的md笔记必须包含以下四个部分，严格按此顺序：

### 1. 一句话概括

用一句中文概括全文核心贡献，格式：**[论文标题]：[一句话概括]**。简洁有力，突出最核心的创新点。

### 2. 背景与问题

- **领域背景**：2-3句话介绍该领域的发展现状和趋势
- **现有不足**：指出已有方法的关键缺陷或未解决的问题
- **本文问题**：明确本文要解决的具体问题，与现有不足直接对应

### 3. 方法介绍

按论文实际结构介绍核心方法，要求：
- 先给出方法的整体框架或核心思路（1-2句话）
- 然后分模块/组件介绍关键技术细节，每个模块用1-3句话
- 关键术语保留英文，必要时用括号附中文解释
- 如果论文有多个变体或阶段，按顺序介绍

### 4. 实验设定与效果

- **数据集与评测指标**：列出使用的数据集和主要评测指标
- **实验结果**：简要概括核心实验结论（与baseline对比的关键数字、是否达到预期效果）
- **分析洞察**：如有消融实验或关键分析，简要指出最重要的1-2个发现

## 读取论文的步骤

1. 从用户提供的URL中提取arxiv ID（如 `2506.10516`）
2. 先访问 `https://arxiv.org/abs/[ID]` 获取标题、作者、摘要等元信息
3. 再访问 `https://arxiv.org/html/[ID]` 获取论文全文HTML版本（比PDF更易解析）
4. 若HTML版本不可用，尝试访问 `https://arxiv.org/pdf/[ID]` 读取PDF
5. 通读全文后按上述结构生成笔记

## 输出格式

将笔记写入md文件，文件名格式为 `[arxiv_id]_notes.md`（如 `2506.10516_notes.md`）。默认保存到用户当前工作目录，用户可指定其他路径。

文件开头包含元信息块：

```markdown
# [论文标题]

- arxiv: [arxiv链接]
- 作者: [作者列表]
- 发表时间: [提交日期]
- 领域: [主要学科分类]

## 一句话概括

**[论文标题]：[一句话概括]**

## 背景与问题

[内容]

## 方法介绍

[内容]

## 实验设定与效果

[内容]
```

## 风格要求

- 简洁为主，避免大段复述论文原文
- 概括而非翻译，用自己的语言提炼要点
- 技术细节点到为止，不展开公式推导
- 实验结果只报关键数字，不列全表
